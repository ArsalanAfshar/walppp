# استودیو موکاپ والپیپر (Walp Mockup Studio)

یک برنامه‌ی مستقل ویندوزی که تصاویر والپیپر را به‌صورت دسته‌ای پردازش
می‌کند (برش، تغییر اندازه، تغییر نام، خروجی) و برای هر کدام به‌طور خودکار
یک موکاپ فتوشاپ می‌سازد؛ این کار با یک اسکریپت ExtendScript (فایل
`.jsx`) که از طریق `subprocess` اجرا می‌شود انجام می‌گیرد.

این پروژه *رفتار* ابزار مرجع `walp2` (خروجی جفت موبایل/دسکتاپ با نام‌گذاری
مبتنی بر کد) را از صفر بازنویسی کرده است — هیچ کدی از آن مخزن وارد یا
استفاده‌ی مجدد نشده — و یک لایه‌ی خودکارسازی فتوشاپ نیز به آن اضافه شده
است.

---

## ۱. این برنامه چه کاری انجام می‌دهد

۱. شما یک یا چند مورد والپیپر به صف اضافه می‌کنید: یک تصویر موبایل، یک
   تصویر دسکتاپ و یک کد کوتاه (مثلاً `0042`).
۲. برای هر مورد، برنامه:
   - تصویر موبایل را برش مرکزی داده و بر اساس کیفیت انتخابی (2K یا 4K)
     تغییر اندازه می‌دهد.
   - همین کار را برای تصویر دسکتاپ انجام می‌دهد.
   - هر دو را در مسیر `OUTPUT_FOLDER\<code>\` با فرمت نام‌گذاری انتخابی
     شما ذخیره می‌کند.
۳. اگر تولید موکاپ فعال باشد، برنامه سپس فتوشاپ را با یک ExtendScript
   تولیدشده اجرا می‌کند که:
   - قالب PSD شما را باز می‌کند.
   - آبجکت هوشمند **"M"** را با خروجی موبایل جایگزین می‌کند.
   - آبجکت هوشمند **"D"** را با خروجی دسکتاپ جایگزین می‌کند.
   - متن بعد از `"Code:"` را در لایه‌ی متنی **"WC"** بازنویسی می‌کند، در
     حالی که فونت، اندازه و سبک آن حفظ می‌شود.
   - رنگ لایه‌ی رنگ‌یکدست **"BC"** را با رنگ RGB/HEX انتخابی شما تغییر
     می‌دهد.
   - هرگز چیزی داخل گروهی که دقیقاً **"1"** نام دارد را تغییر نمی‌دهد.
   - یک موکاپ PNG یا JPG مسطح‌شده خروجی می‌گیرد و فایل را بدون ذخیره روی
     قالب اصلی می‌بندد.

---

## ۲. پیش‌نیازها

- ویندوز ۱۰ یا ۱۱.
- ادوبی فتوشاپ (نسخه‌ی دسکتاپ، هر نسخه‌ی نسبتاً جدید — از CS6 تا آخرین
  نسخه‌های CC همگی از فراخوانی‌های ExtendScript که این برنامه استفاده
  می‌کند پشتیبانی می‌کنند).
- برای *اجرای* فایل `.exe` بسته‌بندی‌شده نیازی به نصب پایتون نیست — کاملاً
  مستقل است. پایتون فقط در صورتی لازم است که بخواهید خودتان آن را از
  سورس بسازید.

---

## ۳. اولین اجرا

اولین باری که `WalpMockupStudio.exe` را اجرا می‌کنید، پنجره‌ای مسیر
`Photoshop.exe` شما را می‌پرسد، برای مثال:

```
C:\Program Files\Adobe\Adobe Photoshop 2024\Photoshop.exe
```

این مسیر در فایل زیر ذخیره می‌شود:

```
%APPDATA%\WalpMockupStudio\config.json
```

می‌توانید بعداً آن را از مسیر **Settings → Photoshop Path...** در پنجره‌ی
اصلی تغییر دهید.

---

## ۴. آماده‌سازی قالب موکاپ شما

کنار فایل `WalpMockupStudio.exe`، پوشه‌ای دقیقاً با این نام بسازید:

```
mockup_template\
```

و یک فایل `.psd` داخل آن قرار دهید. فایل PSD **باید** این لایه‌های دقیق
را داشته باشد (نام‌ها به بزرگی/کوچکی حروف حساس هستند):

| نام لایه | نوع                    | کاربرد                                       |
|----------|------------------------|-----------------------------------------------|
| `M`      | آبجکت هوشمند (Smart Object) | دریافت خروجی والپیپر موبایل             |
| `D`      | آبجکت هوشمند (Smart Object) | دریافت خروجی والپیپر دسکتاپ              |
| `WC`     | لایه‌ی متنی            | باید در جایی از متنش `Code:` داشته باشد؛ هر چه بعد از آن بیاید جایگزین می‌شود |
| `BC`     | لایه‌ی رنگ‌یکدست (Solid Color) | با رنگ پس‌زمینه‌ی انتخابی شما رنگ‌آمیزی می‌شود |
| `1`      | گروه (Layer Set)       | **کاملاً نادیده گرفته می‌شود** — حتی اگر داخلش لایه‌هایی با نام `M`، `D`، `WC` یا `BC` باشد، هرگز تغییر نمی‌کند |

اگر می‌خواهید قالب‌ها را جای دیگری نگه دارید، از فیلد **Template folder**
در پنجره‌ی اصلی استفاده کنید (یا آن را خالی بگذارید تا از پوشه‌ی پیش‌فرض
`mockup_template` کنار فایل اجرایی استفاده شود).

---

## ۵. استفاده از برنامه

۱. **افزودن موارد والپیپر**: روی *Add Item...* کلیک کنید، یک تصویر
   موبایل، سپس یک تصویر دسکتاپ انتخاب کنید، سپس یک کد کوتاه وارد کنید.
   این کار را برای هر تعداد والپیپر که می‌خواهید تکرار کنید — همه در یک
   اجرا پردازش می‌شوند.
۲. **تنظیمات دسته (Batch Settings)**:
   - *Output folder* — مسیر ذخیره‌ی تصاویر پردازش‌شده (و موکاپ‌ها).
   - *Quality* — `2K` (۲۵۶۰×۱۴۴۰ / ۱۴۴۰×۲۵۶۰) یا `4K` (۴۰۹۶×۲۳۰۴ /
     ۲۳۰۴×۴۰۹۶).
   - *Category* — یک کد کوتاه (پیش‌فرض `A` برای انیمه، `O` برای سایر) که
     در نام فایل‌ها گنجانده می‌شود.
   - *Background color* — به‌صورت هگز (`#101010`) یا `R,G,B`
     (`16,16,16`)؛ دکمه‌ی انتخاب‌گر رنگ نیز موجود است.
   - *Code format* — یک الگوی نام فایل با توکن‌های `{prefix}`، `{type}`،
     `{code}`. الگوی پیش‌فرض `{prefix}-{type}-C{code}` نام‌هایی مانند
     `M-A-C0042.jpg` / `D-A-C0042.jpg` تولید می‌کند.
۳. بخش **Photoshop Mockups**:
   - روشن/خاموش کردن *Generate mockups after processing*.
   - انتخاب *Template folder* (اختیاری — پیش‌فرض `mockup_template` است).
   - انتخاب *Export format*: PNG یا JPG.
۴. روی **Run Batch** کلیک کنید. پیشرفت کار و هر خطایی در پنل گزارش سمت
   راست نمایش داده می‌شود. پردازش در یک رشته‌ی پس‌زمینه انجام می‌شود، پس
   پنجره پاسخگو باقی می‌ماند.

نمونه‌ای از ساختار خروجی:

```
OUTPUT_FOLDER\
  0042\
    M-A-C0042.jpg
    D-A-C0042.jpg
    MD-A-C0042-mockup.png
```

## ۶. ساخت فایل exe مستقل به‌صورت دستی

```bat
cd WalpMockupStudio
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
build\build.bat
```

فایل اجرایی نهایی در مسیر `dist\WalpMockupStudio.exe` قرار می‌گیرد. پیش
از توزیع، پوشه‌ی `mockup_template` را کنار آن کپی کنید.

این ساخت از PyInstaller در حالت تک‌فایلی استفاده می‌کند (به `build/app.spec`
نگاه کنید)، بنابراین نتیجه‌ی نهایی هیچ وابستگی خارجی پایتون/DLL فراتر
از آنچه ویندوز از قبل دارد نیاز ندارد.

---

## ۷. نحوه‌ی واقعی کار خودکارسازی فتوشاپ

اسکریپت‌های فتوشاپ نمی‌توانند آرگومان‌های خط فرمان معمولی را مانند یک
برنامه‌ی CLI معمولی دریافت کنند. به همین دلیل این برنامه:

۱. یک فایل `.jsx` کامل تولید می‌کند که تمام مسیرها، رنگ‌ها و مقادیر متنی
   از قبل به‌صورت مقادیر ثابت در آن جای‌گذاری شده‌اند (به
   `app/photoshop/jsx_builder.py` و نمونه‌ی `templates/example_generated_mockup.jsx`
   نگاه کنید).
۲. دستور `subprocess.Popen([photoshop_exe, jsx_path])` را اجرا می‌کند —
   دادن یک فایل `.jsx` به‌عنوان آرگومان فتوشاپ باعث اجرای فوری آن می‌شود،
   دقیقاً مانند دابل‌کلیک روی یک فایل `.jsx` (با ارتباط فایل درست).
۳. اسکریپت برای هر والپیپر یک خط وضعیت به یک فایل متنی کوچک در مسیر
   `%APPDATA%\WalpMockupStudio\temp\` اضافه می‌کند. پایتون آن فایل را
   بررسی می‌کند تا نشانه‌ی `DONE` را ببیند (یا زمان مشخصی سپری شود)، سپس
   موفقیت/شکست هر مورد را در پنل گزارش نمایش می‌دهد.

قالب PSD برای هر والپیپر یک‌بار باز می‌شود، یک نسخه‌ی تکراری از آن مسطح
و خروجی گرفته می‌شود، و هم نسخه‌ی تکراری و هم فایل اصلی **بدون ذخیره**
بسته می‌شوند، بنابراین فایل قالب اصلی شما هرگز تغییر نمی‌کند.

## ۸. رفع اشکال

**«Photoshop executable path is not set or does not exist.»**
از مسیر *Settings → Photoshop Path...* استفاده کرده و آن را به فایل
واقعی `Photoshop.exe` خود اشاره دهید.

**«No .psd file was found in: ...»**
مطمئن شوید یک فایل `.psd` مستقیماً داخل پوشه‌ی `mockup_template` (یا
هر پوشه‌ای که انتخاب کرده‌اید) قرار دارد. فقط باید یک PSD آنجا باشد.

**«Smart Object layer 'M' was not found.» / مشابه برای 'D'، 'WC'، 'BC'**
فایل PSD را در فتوشاپ باز کنید و بررسی کنید نام دقیق و نوع لایه (آبجکت
هوشمند در برابر لایه‌ی معمولی، متنی در برابر شکل و غیره) با جدول بخش ۴
مطابقت دارد. نام لایه‌ها به بزرگی/کوچکی حروف حساس هستند.

**یک کار موکاپ با خطای زمان (timeout) مواجه می‌شود**
معمولاً به این معناست که فتوشاپ یک پنجره‌ی گفتگو نشان داده (هشدار فونت
گم‌شده، پرسش «تغییرات ذخیره شود؟» از یک سند قبلاً بازشده، یا پنجره‌ی
مجوز و مانند آن) که اجرای اسکریپت را مسدود کرده است. پیش از اجرای یک
دسته، هر سند باز دیگری را در فتوشاپ ببندید و مطمئن شوید فایل PSD قالب
جای دیگری باز نیست.

**موکاپ‌ها درست به نظر می‌رسند اما رنگ‌ها اشتباه‌اند**
مطمئن شوید لایه‌ی `BC` واقعاً یک لایه‌ی پرکننده‌ی *Solid Color* است
(از مسیر Layer → New Fill Layer → Solid Color)، نه یک لایه‌ی تنظیم
(adjustment) یا لایه‌ای با افکت روکش رنگ — این‌ها از API متفاوتی در
فتوشاپ استفاده می‌کنند.

**فایل اجرایی اجرا نمی‌شود / آنتی‌ویروس آن را مسدود می‌کند**
این یک هشدار اشتباه رایج برای فایل‌های اجرایی ساخته‌شده با PyInstaller
است. یک استثنا برای `WalpMockupStudio.exe` در آنتی‌ویروس خود اضافه کنید،
یا آن را خودتان با مراحل بخش ۶ از سورس بسازید تا بتوانید کدی که آن را
تولید کرده تأیید کنید.

**نیاز دارم مسیر ذخیره‌شده‌ی فتوشاپ یا تنظیمات دیگر را بازنشانی کنم**
فایل `%APPDATA%\WalpMockupStudio\config.json` را حذف کرده و برنامه را
دوباره اجرا کنید — پنجره‌ی اولین اجرا دوباره ظاهر می‌شود.

## ۹. ساختار پروژه

```
WalpMockupStudio/
  main.py                       نقطه‌ی ورود برنامه
  app/
    config.py                   تنظیمات محلی (مسیر فتوشاپ و غیره)
    core/
      constants.py               پیش‌تنظیمات کیفیت، کدهای دسته‌بندی
      color_utils.py              تجزیه و اعتبارسنجی هگز/RGB
      code_format.py               موتور الگوی نام فایل
      image_processor.py           منطق برش/تغییر اندازه/خروجی (بازنویسی‌شده)
    photoshop/
      jsx_builder.py                تولید سورس ExtendScript
      ps_bridge.py                   اجرای فتوشاپ و تجزیه‌ی نتایج
    gui/
      app_window.py                  پنجره‌ی اصلی Tkinter
      first_run_dialog.py             پنجره‌ی درخواست مسیر فتوشاپ
    utils/
      paths.py                        کمک‌کننده‌های مسیر exe/appdata
      logger.py                        کمک‌کننده‌ی ساده‌ی لاگ
  mockup_template/                 فایل PSD خود را اینجا قرار دهید
  templates/
    example_generated_mockup.jsx    نمونه‌ای از یک اسکریپت تولیدشده (مرجع)
  build/
    app.spec                        فایل تنظیمات PyInstaller
    build.bat                       اسکریپت ساخت با یک دستور
  requirements.txt
  README_EN.md
  README_FA.md
```
