# افزونه موکاپ فتوشاپ برای walp2

یک ماژول کاملاً اختیاری و آفلاین (بدون نیاز به اینترنت یا سرویس ابری) که قابلیت
تولید خودکار موکاپ فتوشاپ را به پایپ‌لاین موجود `walp2` اضافه می‌کند — بدون
تغییر در منطق پردازش، تغییر اندازه یا نام‌گذاری فعلی.

## این ماژول چیست؟

برنامه `wallpaper_manager.py` همین الان تصاویر والپیپر را تغییر اندازه می‌دهد،
با سیستم کدگذاری پروژه نام‌گذاری می‌کند و در `output_dir/{code}/` ذخیره می‌کند.
این افزونه **بعد از** موفقیت آن مرحله وارد عمل می‌شود و در صورت فعال بودن،
فتوشاپ را به‌صورت محلی (Local) اجرا می‌کند تا تصاویر موبایل/دسکتاپ، کد والپیپر
و رنگ پس‌زمینه را داخل یک فایل PSD جایگذاری کرده و خروجی PNG/JPG تولید کند.

بدون سرویس ابری. بدون API خارجی. همه‌چیز روی سیستم شما و از طریق یک نسخه
لایسنس‌دار و نصب‌شده‌ی فتوشاپ اجرا می‌شود.

## پیش‌نیازها

- ویندوز
- نرم‌افزار Adobe Photoshop (نسخه دسکتاپ، دارای لایسنس) نصب‌شده روی همان سیستم
- Python 3.8 یا بالاتر (پیش‌نیاز فعلی خود `walp2`)
- یک فایل قالب PSD با دقیقاً همین نام لایه‌ها:

| نام لایه | نوع لایه            | کاربرد                                                        |
|----------|---------------------|----------------------------------------------------------------|
| `M`      | Smart Object        | تصویر والپیپر موبایل                                            |
| `D`      | Smart Object        | تصویر والپیپر دسکتاپ                                            |
| `WC`     | متن (Text)          | باید شامل `Code:XXXX` باشد — فقط مقدار بعد از `Code:` تغییر می‌کند |
| `BC`     | Solid Color fill    | رنگ پس‌زمینه، از طریق رابط کاربری تنظیم می‌شود (HEX یا RGB)      |
| `1`      | گروه (Group)        | کاملاً نادیده گرفته می‌شود — هرگز خوانده یا تغییر داده نمی‌شود      |

## ساختار پوشه‌ها (سورس)

```
walp2/
├── wallpaper_manager.py     # برنامه اصلی — بدون تغییر
├── requirements.txt         # فایل موجود — بدون تغییر
├── requirements-build.txt   # جدید — فقط برای ساخت فایل exe لازم است
├── walp2.spec               # جدید — فایل تنظیمات PyInstaller
├── build.bat                # جدید — ساخت خودکار exe با یک دستور
├── app.ico                  # اختیاری — آیکون دلخواه شما
└── mockup/                  # این افزونه
    ├── __init__.py
    ├── mockup_engine.py      # کنترلر پایتون
    └── photoshop.jsx         # اسکریپت ExtendScript (داخل فتوشاپ اجرا می‌شود)
```

پوشه `mockup/` (و در صورت نیاز به ساخت exe، فایل‌های build) را در ریشه‌ی
پروژه `walp2` خودتان، درست کنار `wallpaper_manager.py` قرار دهید.

## راه‌اندازی

۱. پوشه `mockup/` را داخل ریشه‌ی پروژه `walp2` کپی کنید.

۲. اگر فتوشاپ در مسیرهای پیش‌فرض نصب نشده، مسیر آن را مشخص کنید:

```bat
set PHOTOSHOP_EXE=C:\Program Files\Adobe\Adobe Photoshop 2025\Photoshop.exe
```

۳. طبق فایل `INTEGRATION_GUIDE.md` بخش رابط کاربری اختیاری و اتصال به تابع
   `done()` را به `wallpaper_manager.py` اضافه کنید. این تغییر بسیار کوچک و
   صرفاً افزودنی است — هیچ تابع موجودی تغییر نمی‌کند.

## ساخت فایل exe مستقل

```bat
build.bat
```

همین کافی است — این اسکریپت در صورت نیاز PyInstaller را نصب می‌کند، فرآیند
build را اجرا می‌کند و به‌طور خودکار `mockup/photoshop.jsx` را در مسیر درست
کپی می‌کند.

**ساختار خروجی نهایی** (این چیزی است که باید تحویل داده شود):

```
dist/
└── walp2/
    ├── walp2.exe
    ├── mockup/
    │   └── photoshop.jsx     ← فایل متنی ساده، بدون نیاز به build مجدد قابل ویرایش است
    └── ...فایل‌های اجرایی PyInstaller...
```

کل پوشه‌ی `dist\walp2\` را تحویل دهید — نه فقط فایل `walp2.exe`. برنامه در
زمان اجرا مسیر `photoshop.jsx` را نسبت به محل خودِ exe پیدا می‌کند
(`mockup_engine.app_base_dir()`)، بنابراین این دو باید همیشه کنار هم باشند.

**چرا فایل jsx داخل exe جاسازی نشده؟** PyInstaller امکان جاسازی فایل‌های
داده را دارد، اما در حالت `--onefile` این فایل‌ها در یک پوشه‌ی موقت استخراج
می‌شوند که هر بار اجرا از نو ساخته و در پایان حذف می‌شود — جایی که نمی‌توان
آن را باز کرد یا ویرایش کرد. از آنجا که شناسه‌های Action Manager (برای
ویرایش Smart Object و Solid Color) ممکن است بین نسخه‌های مختلف فتوشاپ کمی
تفاوت داشته باشند، فایل `photoshop.jsx` عمداً به‌صورت یک فایل ساده و قابل
مشاهده کنار فایل exe نگه داشته شده تا در صورت نیاز، بدون build مجدد قابل
اصلاح باشد.

اگر در یک نسخه‌ی نصب‌شده فایل `photoshop.jsx` گم شود، برنامه یک خطای واضح
`MockupError` همراه با لیست تمام مسیرهایی که جستجو کرده نمایش می‌دهد — هیچ‌وقت
به‌صورت بی‌صدا شکست نمی‌خورد، و هیچ تأثیری روی پایپ‌لاین اصلی تغییر اندازه و
نام‌گذاری ندارد، چون آن بخش اصلاً به این فایل وابسته نیست.

## نحوه استفاده

بعد از انجام یکپارچه‌سازی، یک بخش جدید به رابط کاربری اضافه می‌شود:

- **فعال‌سازی تولید موکاپ** — چک‌باکس، به‌طور پیش‌فرض غیرفعال
- **انتخاب فایل PSD** — انتخاب‌گر فایل برای قالب شما
- **رنگ پس‌زمینه** — ورودی HEX (مثلاً `#1A1A2E`)
- **پوشه خروجی موکاپ** — اختیاری؛ در صورت خالی بودن، پیش‌فرض `output/{code}/mockup/` است

برنامه را طبق روال قبلی اجرا کنید. پایپ‌لاین موجود تغییر اندازه/نام‌گذاری
دقیقاً مثل قبل و اول اجرا می‌شود. فقط در صورت موفقیت آن **و** فعال بودن
چک‌باکس موکاپ، فتوشاپ در پس‌زمینه برای ساخت موکاپ اجرا می‌شود.

### استفاده برنامه‌نویسی (بدون رابط کاربری)

```python
from mockup.mockup_engine import generate_mockups_for_batch

results = generate_mockups_for_batch(
    psd_path=r"C:\templates\mockup.psd",
    batch=[
        {
            "code": "0123",
            "mobile_image": r"C:\output\0123\M-A-C0123.jpg",
            "desktop_image": r"C:\output\0123\D-A-C0123.jpg",
        }
    ],
    mockup_output_dir=r"C:\output\0123\mockup",
    bg_color="#1A1A2E",       # یا (26, 26, 46) یا {"r":26,"g":26,"b":46}
    export_format="png",      # یا "jpg"
    log_callback=print,
)

for r in results:
    print(r["code"], r["status"], r.get("message"), r.get("output"))
```

## نحوه‌ی عملکرد داخلی

۱. `mockup_engine.py` پیش از هرگونه تماس با فتوشاپ، تمام ورودی‌ها را
   اعتبارسنجی می‌کند (وجود فایل‌ها، معتبر بودن رنگ).

۲. یک فایل job به‌صورت JSON در پوشه‌ی temp سیستم‌عامل می‌نویسد.

۳. با استفاده از `subprocess`، دستور `Photoshop.exe photoshop.jsx` را اجرا
   می‌کند — یک نشست فتوشاپ کل دسته (batch) را پردازش می‌کند.

۴. `photoshop.jsx` برای هر آیتم یک **کپی تازه** از فایل PSD باز می‌کند،
   لایه‌های `M`، `D`، `WC` و `BC` را ویرایش می‌کند، موکاپ را خروجی می‌گیرد و
   سند را با `DONOTSAVECHANGES` می‌بندد. فایل قالب اصلی روی دیسک هرگز تغییر
   نمی‌کند.

۵. یک فایل نتیجه (result) به‌صورت JSON می‌نویسد؛ `mockup_engine.py` منتظر آن
   می‌ماند و گزارش موفقیت/شکست هر آیتم را برمی‌گرداند.

## مدیریت خطاها

- چک‌باکس موکاپ غیرفعال باشد ← این ماژول اصلاً اجرا نمی‌شود؛ هیچ تغییری در
  رفتار برنامه ایجاد نمی‌شود.
- مسیر PSD نامعتبر / رنگ نامعتبر / فتوشاپ پیدا نشود ← پیش از اجرای فتوشاپ،
  در سمت پایتون تشخیص داده می‌شود.
- لایه `M` یا `D` وجود نداشته باشد ← فقط همان آیتم با پیام مشخص شکست
  می‌خورد؛ بقیه آیتم‌های دسته همچنان پردازش می‌شوند.
- لایه `WC` یا `BC` وجود نداشته باشد ← فقط یک هشدار ثبت می‌شود، موکاپ همچنان
  خروجی گرفته می‌شود.
- گروه `"1"` ← هرگز پیمایش نمی‌شود؛ این مورد در کد به‌صورت ثابت نادیده گرفته
  شده است.
- هنگ کردن فتوشاپ یا باز شدن یک پنجره‌ی مسدودکننده ← Timeout سمت پایتون
  به‌جای هنگ نامحدود، خطا برمی‌گرداند.
- هر خطای غیرمنتظره در حین ویرایش یک PSD ← در `photoshop.jsx` گرفته می‌شود؛
  سند بدون ذخیره بسته می‌شود تا قالب اصلی هرگز خراب نشود.

## یک نکته مهم درباره تست

این ماژول بر اساس سورس واقعی پروژه `walp2` نوشته شده، اما بدون دسترسی به
نسخه‌ی لایسنس‌دار فتوشاپ یا فایل PSD واقعی شما تست اجرایی نشده است. دستورات
Action Manager استفاده‌شده (جایگزینی Smart Object، تغییر رنگ Solid Color)
تکنیک‌های استاندارد و مستندشده‌ی Adobe هستند و باید روی نسخه‌های اخیر فتوشاپ
کار کنند، اما توصیه می‌شود پیش از اجرای یک batch بزرگ، ابتدا با یک والپیپر
تست شود — در صورت بروز خطا، راهنمای عیب‌یابی در `INTEGRATION_GUIDE.md`
توضیح می‌دهد که چگونه با ضبط یک Action در فتوشاپ، شناسه‌های دقیق مربوط به
نسخه‌ی خودتان را پیدا کنید.

## نکته درباره‌ی وابستگی

طبق مشخصات پروژه، فتوشاپ یک وابستگی الزامی و بدون جایگزین برای این قابلیت
است — اگر روی سیستم اجراکننده نصب و لایسنس‌دار نباشد، هیچ مسیر جایگزینی برای
رندر موکاپ وجود ندارد.
